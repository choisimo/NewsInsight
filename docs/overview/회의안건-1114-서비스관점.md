# 회의 안건 (서비스 관점) – 2025-11-14

## 1. 전체 서비스 로드맵 점검

- 우리가 제공할 핵심 가치가 무엇인지 정리가 되었나요?
- 새 기능을 추가할 때 어떤 서비스부터 우선순위를 둘지 공감대가 있나요?

##  2. API Gateway 서비스 방향
- 외부 고객이 가장 자주 쓰는 경로가 무엇인지, 해당 경로의 성능 목표가 정해져 있나요?
- 인증·권한 정책을 단순화하거나 자동화할 계획이 있나요?
- 장애가 발생했을 때 알림과 대응 흐름이 사용자 관점에서 충분히 빠른가요?

## 3. Collector 서비스 성장 전략

- 새 뉴스 소스를 추가하는 절차가 비기술 직군도 이해할 수 있을 만큼 단순한가요?
- 수집량이 크게 늘어날 경우를 대비한 처리 전략(확장, 우선순위 등)이 준비되어 있나요?
- 수집 결과를 다른 서비스와 어떻게 공유할지에 대한 명확한 경로가 있나요?

## 4. 공용 라이브러리 및 설정 관리

- 여러 서비스에서 공유하는 라이브러리나 설정이 업데이트될 때 영향 범위를 빠르게 파악할 수 있나요?
- 설정을 변경할 때 검토/승인 절차가 명확한가요?
- 문서화나 예제가 부족해서 협업이 느려지는 부분이 있는지 점검이 필요할까요?

## 5. 운영·지원 관점의 공통 과제

- 서비스 현황을 한눈에 볼 수 있는 대시보드나 리포트가 준비되어 있나요?
- 주간/월간 회고에서 반복적으로 등장하는 문제(배포, 장애, 문의 등)가 있다면 해결 계획이 마련되었나요?
